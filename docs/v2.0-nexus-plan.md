> 偏离了项目的真正优势——AI 原生。
>
> v1.0 的资产保留。以下为新方向。

## 三、v2.0 路线图

### Phase 1：AI 辅助写作（现在）

用户在使用 TipTap 编辑器时，AI 实时辅助：

- **AI 续写**：光标位置续写下一段，保持风格一致
- **AI 润色**：选中文本一键优化表达
- **AI 摘要**：文章写完后自动生成摘要和标签建议
- **实现方式**：Next.js API Route 代理 → DeepSeek V4 API（成本极低，约 ¥0.01/次调用）

技术要点：
- TipTap 已有 extension 机制，通过自定义 extension 接入 AI
- 流式输出（streaming）改善体验
- DeepSeek 缓存命中率高，实际成本极低

### Phase 2：语义搜索（本周）

替换当前的 `SQLite LIKE` 搜索：

- **Embedding 向量搜索**：用 DeepSeek embedding API 将文章转为向量
- **语义理解**：用户搜"怎么部署到服务器"，能匹配到标题为"生产环境部署指南"的文章
- **实现成本**：DeepSeek embedding 极便宜，几千篇文章的向量存储用 SQLite 足够

### Phase 3：用户主页 + 互动（本月）

从 Phase 2 回滚中恢复，但严格 Schema 先行：

- 用户主页 `/u/[username]`
- 点赞系统
- 关注/粉丝关系
- **教训**：先 `prisma migrate` 确认 Schema 正确，再写前端

### Phase 4：AI 内容审核（本月）

- 新帖子/评论自动审核（不只是关键词，是语义级判断）
- 垃圾检测、辱骂检测、广告识别
- 人工审核作为兜底（当前的三级权限 + 审核流程保留）

### Phase 5：AI 个性化推荐（下季度）

- 基于用户阅读历史的个性化内容推荐
- 相关文章自动关联
- **关键**：用户可控——能看见为什么推荐了这篇文章，能关闭

### Phase 6：通知系统（按需）

- 评论回复通知
- 关注者新文章通知
- 系统公告

## 六、前一团队的教训（继承）

来自 FAREWELL.md 和 LAST-WORDS.md 的核心经验：

1. **Schema 先行**——先跑通 `prisma migrate`，确认迁移完整，再写前端。Phase 2 回滚的根本原因。
2. **不要跳过架构师**——Team Lead 亲自写代码导致架构偏差。
3. **不要超过 5 个并行 Agent**——9 个 Agent 同时跑，Leader 等不及自己干，半成品堆积。
4. **安全审查不能拖**——Security-Engineer 发现 1 个 CRITICAL + 3 个高危，下次安全审查要在开发阶段并行走。
5. **QA 不是最后一步**——测试应该在开发过程中持续跑，不是等全部写完再测。
